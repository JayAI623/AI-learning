# Positional Encoding Visualizer: 交互设计与开发计划

## 1. 设计初衷与受众定位

**目标群体**：没有编程基础的初中生。
**核心痛点**：大模型就像一个“没有时间观念”的瞎子，它能同时看到所有词，但不知道谁先谁后。如果不加位置编码，“狗咬人”和“人咬狗”对它来说是一模一样的。
**设计准则**：

- 零数学公式优先：不用正余弦公式来吓人，用**“时钟”、“齿轮”和“指南针”**等物理实体来隐喻。
- Apple 级审美：大面积留白，毛玻璃质感，丝滑的动画过渡。
- 引导式探索：不要一上来就展示所有面板，而是通过故事线（Storytelling）一步步解锁。

---

## 2. 核心物理意义与隐喻设计 (The Metaphors)

我们将对比三种状态，层层递进：

### Stage 1: 盲人摸象 —— 没有位置编码 (No PE)

- **物理隐喻**：悬浮在空中的“磁吸单词贴”。
- **可视化效果**：输入一句话（如“The cat chased the mouse”），这5个词像台球一样在屏幕中央随机碰撞、漂浮。
- **物理意义**：展现 Transformer 的本质特征——注意力机制是“词袋模型”（Bag of Words），对顺序完全免疫。

### Stage 2: 挂上时钟 —— 绝对位置编码 (Absolute/Sinusoidal PE - 经典 Transformer)

- **物理隐喻**：给每个词挂上一组“不同转速的时钟”。
- **可视化效果**：
  - 单词排成一排。
  - 每个单词下方挂着 3 个仪表盘（分别代表高频、中频、低频维度）。
  - 第 1 个词：所有指针指向 12 点。
  - 第 2 个词：快指针转了 90 度，中指针转 10 度，慢指针几乎没动。
  - 第 N 个词：指针呈现出独一无二的组合。
- **物理意义**：通过指针的不同角度组合，给每个位置打上**独一无二的条形码**。模型通过看表，就能知道这个词在第几个位置。

### Stage 3: 旋转罗盘 —— 旋转位置编码 (RoPE - 现代大模型如 Llama/Qwen 标配)

- **物理隐喻**：站在同一个罗盘上，关注“相对夹角”而不是“绝对时间”。
- **可视化效果**：
  - 一个巨大的 2D 罗盘。
  - “Cat”在位置 2，被旋转了 20 度。“Mouse”在位置 5，被旋转了 50 度。
  - 它们之间的夹角永远是 30 度（相对距离 5 - 2 = 3）。
  - 用户拖动滑动条，把这句话整体往后移（变成了位置 12 和 15），虽然绝对角度变了，但**它们在罗盘上的相对夹角依然是 30 度！**
- **物理意义**：完美解释现代 LLM 的核心智慧——词与词之间的“相对距离”（你在我后面第几个词）比“绝对位置”（你在本本章的第几个词）重要得多。

## 3. 界面布局与交互流程 (UI/UX Flow)

采用分屏或多 Tab 导航的结构，保持界面极简：

- **顶部导航区**：
  - Segment Control：[The Problem] | [Absolute PE] | [Rotary PE (RoPE)]
- **中央互动区 (Canvas)**：
  - **输入框**：允许用户输入一个短句（最多 6-8 个词），例如 "AI is transforming the modern world"。
  - **可视化舞台**：根据顶部 Tab 切换不同的 SVG/Canvas 动画。
- **底部控制区 (Controls)**：
  - **播放/暂停按钮**：控制动画。
  - **位置滑动条 (Position Slider)**：(RoPE专属) 拖动滑块，可以让整个句子向后平移，观察相对夹角不变的奇妙现象。
  - **维度选择器 (Dimension Toggle)**：切换查看高速齿轮（低维度）和低速齿轮（高维度）。

## 4. 视觉与代码规范 (Technical & Design Specs)

### 视觉规范 (Apple Style)

- **色彩调色板**：
  - 背景：`#F5F5F7` (极简灰底) 或 `#000000` (深邃黑，带纯粹的对比度)。
  - 卡片：`rgba(255, 255, 255, 0.7)` 加 `backdrop-filter: blur(20px)` (毛玻璃效果)。
  - 强调色：
    - 词 A (主语)：`#007AFF` (Apple Blue)
    - 词 B (宾语)：`#FF3B30` (Apple Red)
    - 连线/夹角：`#34C759` (Apple Green)
- **字体**：系统默认无衬线字体 (San Francisco / `system-ui`)，字号加大，标题加粗。

### 技术选型 (Tech Stack)

- **HTML/CSS/JS**：原生实现，保持与之前项目一致，零配置直接在浏览器中打开。
- **D3.js 或 纯 SVG+CSS 动画**：
  - 推荐使用纯 HTML/SVG 结合 CSS `transform: rotate()` 实现，性能极佳且代码易读。
  - 不使用复杂的 React/Vue，保持文件结构简单：
    - `index.html`
    - `css/page.css`
    - `js/app.js` (含位置计算逻辑)
    - `js/svg-builder.js` (自动生成时钟/罗盘图形)

## 5. 开发步骤 (Milestones)

- [ ] **Step 1: 基础框架搭建**
  - 创建目录 `positional-encoding-visualizer`。
  - 搭建基本的 HTML 骨架，引入你已有的 `tokens.css` 和全局样式。
  - 实现顶部的 Tab 切换逻辑。

- [ ] **Step 2: 核心引擎 - Tokenizer 模拟器**
  - 写一个简单的 JS 函数，把用户输入的句子按空格切分成 Token 数组。
  - 给每个 Token 分配初始的颜色和 ID。

- [ ] **Step 3: 动画实现 - 无位置编码 (No PE)**
  - 利用简单的 requestAnimationFrame 制作重力/碰撞或者随机漂浮效果。

- [ ] **Step 4: 动画实现 - 绝对位置编码 (Absolute PE)**
  - 绘制“时钟” SVG 组件。
  - 将 Token 与时钟绑定，根据 Token 的 Index (0, 1, 2...) 计算不同维度的旋转角度并赋给 SVG。

- [ ] **Step 5: 动画实现 - 旋转位置编码 (RoPE)**
  - 绘制大号的“极坐标罗盘”。
  - 将选中的两个 Token 映射到罗盘圆环上，用高亮扇形标注它们之间的“相对夹角”。
  - 增加底部 Slider，实现位置偏移的动态重绘。

- [ ] **Step 6: UI 润色与教育性文案**
  - 添加气泡提示（Tooltips），用大白话解释当前动画的物理意义。
  - 添加平滑的转场动画。
